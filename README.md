## 🛠️ Normalização e Compatibilidade

Para garantir que o repositório funcione de forma consistente em qualquer sistema operacional (macOS, Windows, Linux) e no GitHub:
1. **Nomes de Arquivos e Pastas:** Todos os arquivos e pastas tiveram sua codificação Unicode normalizada. Acentuações e caracteres especiais complexos (como `ç`, `á`, `õ`) foram removidos para evitar links corrompidos ou erros de codificação em diferentes sistemas de arquivos, preservando a capitalização original (ex: `Acao Civil Publica (Lei 7347).docx`).
2. **Formato Atualizado:** Arquivos antigos no formato `.doc` foram convertidos em lote para o padrão moderno `.docx`.
3. **Índice Completo:** Na raiz do projeto encontra-se o arquivo [index.json](index.json) que fornece a lista indexada de toda a árvore de diretórios para ferramentas de busca e scripts automatizados.
