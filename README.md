## 🎯 Metodologia de Estudo Recomendada

Para obter o melhor rendimento do acervo de legislação grifada:

1. **Anotações Otimizadas:** Utilize leitores que suportem marcações em `.docx` (como MS Word, Pages ou LibreOffice) para personalizar seus grifos com novas cores ou inserir notas de jurisprudência (Informativos STF/STJ) diretamente ao texto da lei.
2. **Leitura Ativa:** O texto da lei seca já contém as partes fundamentais destacadas. Concentre-se nas palavras-chave, prazos e exceções que costumam ser cobrados em provas.
3. **Sincronização:** Clone este repositório em sua máquina local ou tablet para ter o material de estudo offline sempre à mão.

## 🛠️ Detalhes de Infraestrutura e Normalização

Este repositório foi construído visando robustez técnica e facilidade de pesquisa:

* **Compatibilidade Unicode:** Todos os arquivos foram normalizados de NFD (comum no macOS) para NFC. Acentos e caracteres especiais nos nomes de arquivos e pastas foram substituídos por suas formas equivalentes (ex: `é` -> `e`, `ç` -> `c`) para evitar links e caminhos quebrados ao transferir arquivos entre computadores ou cloná-los no Windows.
* **Formatos Modernos:** Todos os arquivos antigos em formato `.doc` foram convertidos para `.docx` usando a ferramenta de linha de comando `textutil` do macOS, melhorando o carregamento e reduzindo o consumo de espaço em disco.
* **Estrutura Git Otimizada:** O repositório contém um arquivo `.gitignore` que impede que arquivos de sistema (como `.DS_Store` do macOS) ou arquivos temporários do Word (como `~$*.docx`) sejam commitados.

## 📈 Como Buscar no Repositório

O repositório disponibiliza um arquivo de índice centralizado: **[index.json](index.json)**.
Ele contém a árvore completa e atualizada dos diretórios. Você pode usar este arquivo para:
- Executar scripts de automação ou busca rápida.
- Verificar quais leis estão presentes sem precisar baixar todo o acervo.
- Fazer buscas textuais rápidas locais na sua máquina usando editores como VS Code ou utilitários JSON (`jq`).

## 🤝 Como Contribuir

Contribuições para correções e atualizações são muito bem-vindas!
Se você encontrar alguma lei desatualizada ou com formatação corrompida:
1. Faça um **Fork** deste repositório.
2. Crie uma branch para a sua alteração (`git checkout -b feature/atualizacao-lei`).
3. Faça o commit da sua alteração (`git commit -m "Atualiza Lei XYZ conforme alteração legislativa recente"`).
4. Faça o **Push** para a branch (`git push origin feature/atualizacao-lei`).
5. Abra um **Pull Request**.
